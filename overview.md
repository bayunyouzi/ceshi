# GPT 模式直连/CORS 报错修复概览

## 问题原因
- 用户看到的提示：`无法直连该自定义生图接口...CORS...`。
- 代码复查后确认：GPT-Image-2 文生图本身已经是请求本站后端 `/api/generate-image`，不是浏览器直连 `yzgpt`。
- 但前端错误提示仍沿用了“自定义接口直连/CORS”的旧文案，所以 GPT 模式下只要浏览器访问本站后端失败，也会误报成自定义接口/CORS。
- 另外 GPT 模式下的“生成提示词”路径仍可能因为用户填写过自定义提示词接口而走浏览器直连；这会把 `gpt-5.5` 提示词接口暴露给浏览器侧，容易遇到 CORS 或网络失败。

## 已修复
- `app/page.tsx` 中，GPT 模式下的提示词生成现在强制走本站后端 `/api/generate` 代理，不再浏览器直连 `yzgpt`。
- GPT 生图失败时，如果是浏览器到本站后端的连接失败，错误文案改成：`无法连接本站 GPT 生图后端，请确认 Zeabur 服务已重新部署并正常运行。`
- Grok 自定义接口逻辑保留原样：只有非 GPT 模式的自定义接口才继续使用浏览器直连提示。

## 验证结果
- `npm run build` 构建成功。

- `app/page.tsx`

## 部署注意
- 这次是前端逻辑和错误文案修复，需要重新部署 Zeabur 后才会在线上生效。
- 如果线上仍显示旧的 CORS 文案，说明访问到的仍是旧构建，需要等待 Zeabur 完成构建并清缓存/刷新页面。
